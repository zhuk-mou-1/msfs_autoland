# Auto-Detect: Автоматическое определение захода из MSFS

## Обзор

Функция **Auto-Detect** автоматически считывает параметры захода напрямую из Microsoft Flight Simulator в реальном времени через SimConnect API.

## Как это работает

### Что определяется автоматически

1. **ILS заход** (если настроен NAV1):
   - Частота ILS (MHz)
   - Курс localizer (градусы)
   - Координаты порога ВПП
   - Превышение ВПП

2. **GPS заход** (если установлен пункт назначения):
   - Координаты порога ВПП
   - Превышение ВПП

3. **Общие параметры**:
   - Угол глиссады (3.0° по умолчанию)
   - Высота принятия решения
   - Скорость захода

## Использование

### В GUI

1. Запустите MSFS и загрузите полёт
2. **Настройте ILS в симуляторе:**
   - Откройте радиопанель (обычно G1000/G3000 или отдельная панель)
   - Настройте NAV1 на частоту ILS нужной ВПП
   - Убедитесь что localizer захвачен (зелёный индикатор)
   
   **ИЛИ**
   
   **Установите пункт назначения:**
   - Откройте GPS/FMC
   - Установите аэропорт назначения
   - Выберите ВПП

3. Запустите `python gui.py`
4. Нажмите "Connect"
5. Нажмите "Start Approach"
6. Выберите вкладку **"Auto-Detect"**
7. Нажмите **"Detect Approach"**
8. Проверьте определённые параметры
9. Нажмите **"OK"** для начала захода

### Пример вывода

## Требования

### Для ILS захода

**В MSFS должно быть настроено:**
1. NAV1 радио на частоте ILS
2. Самолёт должен быть в зоне действия ILS (обычно 25nm)
3. Localizer должен быть доступен

**Как настроить в популярных самолётах:**

**Cessna 172 (G1000):**
1. Нажмите на NAV1 частоту на PFD
2. Введите частоту ILS (например, 110.30)
3. Нажмите ENT

**Boeing 737:**
1. Откройте радиопанель (overhead panel)
2. Настройте NAV1 на частоту ILS
3. Установите курс на MCP

**Airbus A320:**
1. Откройте MCDU
2. RADIO NAV page
3. Введите частоту ILS в NAV1

### Для GPS захода

**В MSFS должно быть настроено:**
1. Пункт назначения в GPS/FMC
2. Выбрана ВПП

## Что НЕ определяется автоматически

- Длина ВПП (используется 8000 ft по умолчанию)
- Ширина ВПП (используется 150 ft по умолчанию)
- Название аэропорта/ВПП

Эти параметры не критичны для выполнения захода, но влияют на расчёт посадочной дистанции.

## Преимущества Auto-Detect

✅ **Быстро** - не нужно искать в базе данных  
✅ **Точно** - данные напрямую из симулятора  
✅ **Актуально** - всегда соответствует текущей настройке MSFS  
✅ **Просто** - один клик для определения  

## Сравнение методов

| Метод | Скорость | Точность | Полнота данных |
|-------|----------|----------|----------------|
| **Auto-Detect** | ⚡ Быстро | ✅ Высокая | ⚠️ Базовая |

## Технические детали

### SimConnect переменные

**Для ILS:**
```python
NAV_HAS_LOCALIZER:1      # Наличие localizer
NAV_ACTIVE_FREQUENCY:1   # Частота (Hz)
NAV_LOCALIZER:1          # Курс localizer (градусы)
```

**Для GPS:**
```python
GPS_WP_NEXT_LAT          # Широта следующей точки
GPS_WP_NEXT_LON          # Долгота следующей точки
GPS_WP_NEXT_ALT          # Высота следующей точки
```

**Текущая позиция:**
```python
PLANE_LATITUDE           # Широта самолёта
PLANE_LONGITUDE          # Долгота самолёта
PLANE_ALTITUDE           # Высота самолёта
PLANE_HEADING_DEGREES_MAGNETIC  # Магнитный курс
```

### Алгоритм определения

```python
def auto_configure_approach():
    # 1. Проверка ILS
    if NAV_HAS_LOCALIZER:
        frequency = NAV_ACTIVE_FREQUENCY
        course = NAV_LOCALIZER
        return ILS_approach(frequency, course)
    
    # 2. Проверка GPS
    if GPS_WP_NEXT_LAT and GPS_WP_NEXT_LON:
        lat = GPS_WP_NEXT_LAT
        lon = GPS_WP_NEXT_LON
        return GPS_approach(lat, lon)
    
    # 3. Ничего не найдено
    return None
```

## Устранение проблем

### "No approach data detected"

**Причины:**
1. NAV1 не настроен на ILS частоту
2. Нет пункта назначения в GPS
3. Самолёт слишком далеко от ILS передатчика
4. ILS не доступен на данной ВПП

**Решение:**
1. Проверьте настройку NAV1 радио
2. Убедитесь что вы в зоне действия ILS
3. Установите пункт назначения в GPS
4. Используйте "From Database" или "Manual Entry"

### Неправильная частота

**Причина:** NAV1 настроен на другую частоту

**Решение:**
1. Проверьте частоту ILS для вашей ВПП
2. Перенастройте NAV1
3. Повторите Auto-Detect

### Неправильный курс

**Причина:** Localizer передаёт курс для противоположной ВПП

**Решение:**
1. Убедитесь что настроили правильную частоту
2. Проверьте направление захода
3. Используйте "From Database" для точных данных

## Примеры использования

### Пример 1: ILS заход в Шереметьево 07C

**В MSFS:**
1. Настройте NAV1: 110.30 MHz
2. Установите курс: 070°

**В программе:**
1. Auto-Detect → Detect Approach
2. Результат:
   ```
   Type: ILS
   Frequency: 110.30 MHz
   Course: 70°
   ```
3. OK → Начало захода

### Пример 2: GPS заход

**В MSFS:**
1. Откройте GPS
2. Direct To → UUEE
3. Выберите ВПП 07C

**В программе:**
1. Auto-Detect → Detect Approach
2. Результат:
   ```
   Type: GPS
   Runway Threshold: 55.9728, 37.4106
   ```
3. OK → Начало захода

## Ограничения

- Не определяет название аэропорта/ВПП
- Не определяет точную длину/ширину ВПП
- Требует предварительной настройки в MSFS
- Работает только при активном подключении к MSFS

## Рекомендации

**Используйте Auto-Detect когда:**
- Вы уже настроили ILS в симуляторе
- Нужно быстро начать заход
- Аэропорта нет в базе данных

**Используйте From Database когда:**
- Нужны точные параметры ВПП
- Планируете заход заранее
- Хотите видеть все доступные заходы

**Используйте Manual Entry когда:**
- Нестандартные параметры
- Тестирование
- Аэропорта нет в базе и MSFS
