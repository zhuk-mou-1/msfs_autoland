# Автоматическое выравнивание при посадке (Flare)

## Обзор

Выравнивание (flare) - это критически важная фаза посадки, когда самолёт плавно переходит от снижения по глиссаде к горизонтальному полёту непосредственно перед касанием ВПП.

## Что такое выравнивание?

Выравнивание - это манёвр, при котором:
1. Плавно увеличивается тангаж (2-3° → 5-7°)
2. Снижается вертикальная скорость (600 fpm → 100 fpm)
3. Самолёт удерживается над ВПП 1-2 секунды
4. Происходит мягкое касание с VS ~100-200 fpm

## Параметры выравнивания

### Высоты

- **Начало выравнивания**: 30 футов AGL
- **Конец выравнивания**: 5 футов AGL
- **Касание**: 2-3 фута AGL

### Тангаж

- **Начальный** (на глиссаде): 2.5°
- **Целевой** (при касании): 6.0°
- **Максимальная скорость изменения**: 2.0°/сек

### Вертикальная скорость

- **Начальная** (на глиссаде): -600 fpm
- **Целевая** (при касании): -100 fpm

### Газ

- **Начало сброса**: 50 футов
- **Минимальный газ**: 5% (idle)

## Алгоритм выравнивания

### Фаза 1: Определение момента начала (30ft)

```python
if radio_height <= 30.0 and vertical_speed < 0:
    start_flare()
```

### Фаза 2: Расчёт прогресса

Прогресс выравнивания рассчитывается по высоте:

```
progress = 1.0 - ((radio_height - 5.0) / (30.0 - 5.0))
progress = 0.0 (начало) → 1.0 (конец)
```

### Фаза 3: Плавное изменение тангажа

Используется квадратичная функция для плавности:

```
target_pitch = 2.5° + (6.0° - 2.5°) × progress^0.7
```

**Кривая тангажа:**
- 30ft: 2.5°
- 20ft: 3.5°
- 10ft: 5.0°
- 5ft: 6.0°

### Фаза 4: Контроль вертикальной скорости

```
target_vs = -600 + (-100 - (-600)) × progress^0.5
```

**Кривая VS:**
- 30ft: -600 fpm
- 20ft: -400 fpm
- 10ft: -200 fpm
- 5ft: -100 fpm

### Фаза 5: Управление газом

```
if radio_height < 50ft:
    throttle_progress = 1.0 - (radio_height / 50.0)
    throttle = 0.3 × (1 - throttle_progress) + 0.05 × throttle_progress
```

---

## PD-контроллер тангажа

Для точного управления используется PD-контроллер:

```
error = target_pitch - current_pitch
elevator_input = Kp × error - Kd × pitch_rate

Kp = 0.15 (пропорциональный коэффициент)
Kd = 0.05 (коэффициент демпфирования)
```

## Коррекция на ветер

### Встречный ветер (>15 узлов)

- Начало выравнивания: +5 футов (35ft вместо 30ft)
- Причина: больше времени на выравнивание

### Попутный ветер (>5 узлов)

- Начало выравнивания: -3 фута (27ft вместо 30ft)
- Причина: быстрее приближается земля

## Использование в программе

### Автоматическое выравнивание

Система автоматически выполняет выравнивание в фазе LANDING:

```python
# Выравнивание происходит автоматически
system.start_approach()
system.execute_approach()
```

### Настройка параметров

Можно настроить параметры выравнивания:

```python
from modules.flare_controller import FlareConfig

config = FlareConfig(
    flare_start_height=30.0,    # начало выравнивания
    flare_end_height=5.0,       # конец выравнивания
    initial_pitch=2.5,          # начальный тангаж
    target_pitch=6.0,           # целевой тангаж
    max_pitch_rate=2.0,         # макс. скорость изменения
    initial_vs=-600.0,          # начальная VS
    target_vs=-100.0,           # целевая VS
    throttle_reduction_start=50.0,
    min_throttle=0.05
)

flare_controller = FlareController(config)
```

### Настройка под тип самолёта

**Лёгкие самолёты (Cessna 172):**
```python
config = FlareConfig(
    flare_start_height=25.0,
    target_pitch=7.0,
    initial_vs=-500.0,
    target_vs=-80.0
)
```

**Средние самолёты (Boeing 737):**
```python
config = FlareConfig(
    flare_start_height=35.0,
    target_pitch=5.5,
    initial_vs=-700.0,
    target_vs=-120.0
)
```

**Тяжёлые самолёты (Boeing 777):**
```python
config = FlareConfig(
    flare_start_height=40.0,
    target_pitch=5.0,
    initial_vs=-800.0,
    target_vs=-150.0
)
```

## Типичные проблемы и решения

### Слишком раннее выравнивание

**Проблема:** Самолёт зависает над ВПП
**Решение:** Уменьшить `flare_start_height`

### Слишком позднее выравнивание

**Проблема:** Жёсткое касание
**Решение:** Увеличить `flare_start_height`

### Слишком быстрое изменение тангажа

**Проблема:** Резкие движения
**Решение:** Уменьшить `max_pitch_rate`

### Недостаточный тангаж

**Проблема:** Касание носовым колесом
**Решение:** Увеличить `target_pitch`

## Физика выравнивания

### Ground Effect (эффект экрана)

На высоте менее 1 размаха крыла (~30-50ft) возникает эффект экрана:
- Увеличивается подъёмная сила
- Уменьшается индуктивное сопротивление
- Самолёт "плывёт" над ВПП

Система учитывает это через плавное изменение тангажа.

### Энергетический баланс

При выравнивании:
- Кинетическая энергия → Потенциальная энергия
- Скорость снижения → Высота над ВПП
- Газ снижается для контроля энергии

## Безопасность

### Ограничения

- Максимальный вход руля высоты: ±0.3 (30%)
- Максимальная скорость изменения тангажа: 2°/сек
- Минимальная высота начала: 20 футов
- Максимальная высота начала: 50 футов

### Аварийные ситуации

Если на высоте <10 футов:
- VS > 1000 fpm → возможно жёсткое касание
- Тангаж < 3° → возможно касание носовым колесом
- Тангаж > 10° → возможен tail strike
