## Сценарий тестирования 1: Базовая проверка подключения

### Цель
Проверить подключение к MSFS, определение самолёта, инициализацию систем.

### Шаги

1. **Запустить GUI**
   ```bash
   cd C:\BAT\msfs_autoland
   python gui.py
   ```

2. **Загрузить полёт в MSFS**
   - Самолёт: **TBM 930** (Asobo, стандартный)
   - Аэропорт: **KSEA** (Seattle-Tacoma International)
   - ВПП: **16L** или **16R**
   - Погода: **Clear Skies** (ясная погода)
   - Время: **День** (для лучшей видимости)
   - Позиция: **В воздухе** на высоте 3000ft, 15nm от аэропорта

3. **Нажать "Connect" в GUI**
   - Должно появиться: "Connected to MSFS"
   - Проверить панель "Aircraft & Autopilot":
     - Aircraft: TBM 930 Asobo
     - Profile: Standard MSFS
     - Status: Full Compatibility (зелёный)

4. **Проверить звуковые предупреждения**
   - Должно появиться сообщение: "Audio alerts ready"
   - Время генерации: 5-10 секунд
   - Проверить папку `audio_alerts/` - должно быть 9 MP3 файлов

5. **Проверить телеметрию**
   - Панель "Telemetry" должна показывать:
     - Position (широта, долгота, высота)
     - Attitude (крен, тангаж, курс)
     - Speed (IAS, GS, VS)
   - Данные должны обновляться в реальном времени

6. **Проверить логи**
   ```bash
   # Текстовый лог
   tail -f logs/autoland.log
   
   # Структурированные логи
   ls -la logs/session_*.jsonl
   ls -la logs/logs.db
   ```

### Ожидаемый результат
- ✅ Подключение успешно

## Сценарий тестирования 2: ILS заход на посадку

### Цель
Протестировать автоматический заход на посадку с использованием ILS.

### Подготовка

1. **Настройка полёта в MSFS**
   - Самолёт: **TBM 930**
   - Аэропорт: **KSEA** (Seattle-Tacoma)
   - ВПП: **16L** (ILS 110.30)
   - Позиция: **В воздухе**
     - Высота: **3000 ft MSL**
     - Расстояние: **15 nm** от порога ВПП
     - Курс: примерно **160°** (на ВПП)
     - Скорость: **150 KIAS**
   - Погода: **Clear Skies**, ветер 5-10 kt
   - Конфигурация самолёта:
     - Закрылки: **Takeoff** (1-2 позиция)
     - Шасси: **Убрано**
     - Автопилот: **Выключен** (AutoLand сам включит)

2. **Настройка в GUI**
   - Connect to MSFS
   - Включить чекбокс **"Use ILS"**
   - Включить чекбокс **"Use Autothrottle"**
   - Включить чекбокс **"Audio Alerts"**

3. **Настройка захода**
   - Нажать кнопку **"Auto-Detect from MSFS"**
   - Проверить параметры:
     - ILS Frequency: 110.30
     - Runway Heading: 158°
     - Approach Speed: 90 KIAS
     - Glideslope Angle: 3.0°

### Выполнение захода

1. **Запуск захода**
   - Нажать кнопку **"Start Approach"**
   - Статус должен измениться на: **"INITIAL"**

2. **Фаза INITIAL (перехват курса)**
   - AutoLand включит автопилот
   - Самолёт начнёт разворот на курс посадки
   - Следить за панелью "Navigation":
     - Distance to station (уменьшается)
     - Cross track error (должен стремиться к 0)
     - On course (должен стать True)
   - **Ожидаемое время:** 2-5 минут

3. **Фаза INTERMEDIATE (следование по курсу)**
   - Самолёт следует по курсу ILS
   - Начинается снижение
   - Следить за:
     - Altitude (снижается к глиссаде)
     - Required altitude (целевая высота)
     - Vertical speed (около -500 fpm)
   - **Ожидаемое время:** 3-7 минут

4. **Фаза FINAL (точное следование)**
   - Активируется autothrottle
   - Точное следование по ILS localizer и glideslope
   - Следить за:
     - ILS deviation (должен быть < 1 dot)
     - Glideslope deviation (должен быть < 1 dot)
     - Approach speed (поддерживается автоматически)
   - **Проверка сдвига ветра:**
     - Панель "Wind Shear" должна показывать статус
     - При обнаружении сдвига - звуковое предупреждение
   - **Ожидаемое время:** 5-10 минут

5. **Фаза LANDING (выравнивание)**
   - Начинается на высоте ~50 ft AGL
   - Плавное увеличение тангажа
   - Снижение вертикальной скорости
   - Касание ВПП: -100 to -300 fpm (идеально)
   - **Ожидаемое время:** 10-30 секунд

6. **После посадки**
   - Нажать **"Stop Approach"**
   - Проверить логи

### Что наблюдать

**В GUI:**
- Плавное изменение фаз: INITIAL → INTERMEDIATE → FINAL → LANDING
- Уменьшение distance to station
- Стабильный cross track error (< 2°)
- Поддержание approach speed (±5 KIAS)
- Плавное снижение по глиссаде

**В MSFS:**
- Автопилот включается автоматически
- Самолёт разворачивается на курс
- Плавное снижение
- Точное следование по ILS
- Мягкое касание ВПП

**В логах:**
```bash
# Следить за логами в реальном времени
tail -f logs/autoland.log
```

### Возможные проблемы

**Проблема 1: Самолёт не разворачивается**
- Проверить: автопилот включён?
- Проверить: правильный курс в настройках?
- Решение: вручную развернуть на курс, затем Start Approach

**Проблема 2: Самолёт не снижается**
- Проверить: altitude hold включён?
- Проверить: vertical speed mode работает?
- Решение: вручную установить VS -500 fpm

**Проблема 3: Слишком быстрое снижение**
- Проверить: approach speed правильная?
- Проверить: autothrottle активен?
- Решение: увеличить throttle вручную

**Проблема 4: Промах мимо ВПП**
- Проверить: ILS frequency правильная?
- Проверить: runway heading правильный?
- Решение: выполнить go-around, настроить заново

**Проблема 5: Жёсткая посадка**
- Это нормально для первых тестов
- Flare controller требует настройки
- Решение: записать параметры, настроить позже

## Сценарий тестирования 3: Проверка звуковых предупреждений

### Цель
Протестировать систему звуковых предупреждений о сдвиге ветра.

### Подготовка

1. **Настройка полёта**
   - Самолёт: **TBM 930**
   - Аэропорт: **KSEA**
   - ВПП: **16L**
   - Позиция: **Final approach**, 500 ft AGL
   - Погода: **Custom**
     - Ветер на высоте: 20 kt from 180°
     - Ветер у земли: 5 kt from 270° (создаёт сдвиг)

2. **Настройка в GUI**
   - Connect to MSFS
   - Включить **"Audio Alerts"**
   - Включить **"Use ILS"**

### Выполнение

1. **Запустить заход**
   - Start Approach
   - Дождаться фазы FINAL

2. **Создать сдвиг ветра**
   - В MSFS открыть Weather settings
   - Резко изменить ветер:
     - Скорость: с 5 kt на 25 kt
     - Направление: с 180° на 090°
   - Применить изменения

3. **Наблюдать реакцию**
   - Панель "Wind Shear" должна показать:
     - Type: HEADWIND_LOSS или TAILWIND_GAIN
     - Severity: WARNING или CRITICAL
     - Magnitude: > 10 kt
   - **Должно прозвучать:**
     - WARNING: "WINDSHEAR AHEAD! CAUTION!"
     - CRITICAL: "WINDSHEAR! WINDSHEAR! GO AROUND!"
   - При CRITICAL на высоте < 500 ft:
     - Автоматический go-around
     - Full throttle
     - Climb 1500 fpm

### Ожидаемый результат
- ✅ Обнаружение сдвига ветра
- ✅ Звуковое предупреждение воспроизведено
- ✅ Правильная классификация severity
- ✅ Автоматический go-around при CRITICAL
- ✅ Логирование события

   ```bash

   cursor.execute('SELECT level, COUNT(*) FROM logs GROUP BY level')
   print('By level:')
   for row in cursor.fetchall():
   ```

## Известные ограничения

### Текущие ограничения системы

1. **MobiFlight WASM не работает**
   - Кастомные автопилоты (PMDG, Fenix) имеют ограниченную поддержку
   - Используется fallback на SimConnect
   - Рекомендуется тестировать на стандартных самолётах MSFS

2. **Flare controller требует настройки**
   - Первые посадки могут быть жёсткими
   - Параметры выравнивания зависят от самолёта
   - Требуется калибровка для каждого типа

3. **Autothrottle базовый**
   - PID контроллер требует настройки
   - Может быть неточным на некоторых самолётах
   - Рекомендуется мониторить скорость вручную
