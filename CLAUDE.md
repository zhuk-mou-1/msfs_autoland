# Инструкции для Claude - MSFS AutoLand Project

## О проекте

**MSFS AutoLand System** - система автоматической посадки для Microsoft Flight Simulator 2020/2024 с поддержкой ILS, VOR, NDB и GPS заходов.

**Язык разработки:** Python 3.7+  
**Основные технологии:** SimConnect API, tkinter GUI, MobiFlight WASM  
**Расположение:** C:\BAT\msfs_autoland

## Архитектура проекта

### Основные модули

1. **main.py** - главная система AutoLandSystem
   - Управление фазами захода (IDLE → INITIAL → INTERMEDIATE → FINAL → LANDING)
   - Интеграция всех подсистем
   - Класс AutoLandSystem - центральная точка управления

2. **gui.py** - графический интерфейс
   - Tkinter-based GUI
   - Панели: Telemetry, Aircraft & Autopilot, Navigation, Stabilization, Flare, Logs
   - Обновление в реальном времени (500мс)

3. **modules/telemetry.py** - чтение данных из MSFS
   - SimConnect интеграция
   - Чтение позиции, скорости, ориентации, погоды
   - Определение типа самолёта

4. **modules/aircraft_adapter.py** - адаптер кастомных автопилотов
   - Автоматическое определение типа самолёта
   - Поддержка PMDG, Fenix, FSLabs, FlyByWire
   - Fallback на SimConnect
   - Интеграция с MobiFlight WASM для L:Vars

5. **modules/control.py** - управление самолётом
   - SimConnect команды
   - Управление автопилотом, тягой, рулями

6. **modules/navigation.py** - навигация
   - Расчёт курса, расстояния, отклонений
   - VOR/NDB/ILS навигация

7. **modules/ils_navigation.py** - ILS заходы
   - Чтение localizer и glideslope
   - Точное наведение по ILS

8. **modules/msfs_airport_reader.py** - чтение данных аэропортов из MSFS
   - Автоопределение ILS частоты и курса
   - Чтение GPS пункта назначения
   - Магнитное склонение, угол глиссады

9. **modules/autothrottle.py** - автоматическая тяга
   - PID контроллер скорости
   - Учёт веса и конфигурации
   - Интеграция с vJoy

10. **modules/flare_controller.py** - выравнивание при посадке
    - Плавное увеличение тангажа
    - Снижение вертикальной скорости
    - PD-контроллер

11. **modules/stabilized_approach.py** - мониторинг стабилизации
    - Проверка критериев ICAO/FAA
    - Автоматический уход на второй круг

12. **modules/virtual_joystick.py** - vJoy интеграция
    - Прямое управление рулями
    - Точная коррекция крена и тангажа

13. **modules/fms_reader.py** - чтение FMS/GPS
    - Маршрутные точки
    - Прогресс по STAR

14. **modules/wasm_interface.py** - MobiFlight WASM
    - Чтение/запись L:Vars
    - Отправка кастомных событий
    - CLIENT_DATA коммуникация

### Конфигурационные файлы

- **config/aircraft_profiles.json** - профили автопилотов самолётов
- **config/airports_database.json** - база данных аэропортов (8 аэропортов)

## Важные концепции

### Фазы захода
1. **IDLE** - ожидание
2. **INITIAL** - перехват курса посадки
3. **INTERMEDIATE** - следование по курсу, снижение
4. **FINAL** - точное следование по глиссаде
5. **LANDING** - выравнивание и посадка
6. **COMPLETED** - завершено

### Методы команд автопилота
- **simconnect** - стандартные команды MSFS (работает всегда)
- **lvar** - локальные переменные через WASM (требует MobiFlight)
- **event** - кастомные события через WASM (требует MobiFlight)

### Fallback механизм
Если WASM недоступен или команда не выполнена:
1. Автоматическое переключение на SimConnect
2. Логирование предупреждения
3. Продолжение работы с базовой функциональностью

## Система памяти и контекста (L4 SEMANTIC)

### При каждом запуске сессии

**ОБЯЗАТЕЛЬНО выполни:**

1. Проверь наличие памяти в глобальной директории Claude:
   - `~/.claude/projects/C--BAT-msfs_autoland/memory/handoff.md`
   - `~/.claude/projects/C--BAT-msfs_autoland/memory/decisions.md`
2. Прочитай оба файла
3. Кратко подтверди контекст пользователю

**Пример:**
```
Контекст загружен: WASM интеграция завершена (02:05), кастомные автопилоты поддерживаются
```

### Структура памяти

**HOT Memory (handoff.md)** - последние 10 событий за 24ч
- Автоматически обновляется через stop hook
- Быстрый контекст "что было в последний раз"

**WARM Memory (decisions.md)** - решения за 14 дней
- Архитектурные и технические решения
- Почему выбрали SimConnect, WASM, vJoy
- Что отложено и почему

**COLD Memory (archive/)** - долгосрочное хранение
- Автоматически индексируется в L4 SEMANTIC

**L4 SEMANTIC (semantic_db/)** - векторная БД
- Семантический поиск по всей истории проекта
- Поиск по смыслу: "как мы решали проблему с PMDG" → находит все связанные решения
- Мультиязычный (русский + английский)

### Использование L4 SEMANTIC

**Ручной поиск:**
```bash
l4_search.bat "PMDG autopilot integration"
l4_search.bat "как мы решали проблему с WASM"
```

**Автоматический контекст:**
При использовании триггеров ("как мы", "почему мы", "раньше") автоматически добавляется контекст из L4.

### Автоматические системы

**Stop Hook** - записывает session summary при завершении
**Session Start Hook** - фиксирует timestamp начала
**Semantic Search Hook** - автоматически добавляет контекст из L4
