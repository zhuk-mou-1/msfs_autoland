# MSFS AutoLand System

Система автоматической посадки для Microsoft Flight Simulator с поддержкой ILS, VOR и NDB заходов.

## Возможности

- Подключение к MSFS через SimConnect API
- Чтение телеметрии (позиция, высота, скорость, курс, крен)
- **Радиовысотомер** - точная высота над поверхностью
- **Полная поддержка кастомных автопилотов (NEW!):**
  - Автоматическое определение типа самолёта
  - PMDG 737/777, Fenix A320, FSLabs A32X, FlyByWire A32NX
  - Использование правильных команд для каждого производителя
  - Поддержка L:Vars через MobiFlight WASM
  - Fallback на SimConnect если WASM недоступен
  - Визуализация статуса автопилота в GUI
- **Поддержка кастомных самолётов:**
  - Определение PMDG 737/777, Fenix A320, FSLabs A32X, iniBuilds A300/A310
  - Система профилей для каждого производителя
  - Адаптер команд с fallback на SimConnect
  - Готово к интеграции с FSUIPC/MobiFlight WASM
- **Определение типа самолёта:**
  - Автоматическое определение производителя и модели
  - Классификация автопилота (NONE/LIMITED/BASIC/STANDARD/ADVANCED)
  - Проверка совместимости с AutoLand системой
- **ILS заходы:**
  - Автоматическое чтение localizer и glideslope
  - Точное наведение по курсу и глиссаде
  - Отклонения в градусах и точках индикатора
  - Поддержка стандартных и крутых глиссад
- **Auto-Detect из MSFS:**
  - Автоматическое определение параметров захода из симулятора
  - Чтение ILS частоты и курса из NAV1
  - Чтение GPS пункта назначения
  - Один клик для настройки захода
- **Autothrottle - Автоматическая тяга:**
  - PID контроллер для точного управления скоростью
  - Учёт веса самолёта и конфигурации (закрылки/шасси)
  - Автоматическая коррекция на ветер
  - Интеграция с vJoy для прямого управления
  - Активируется автоматически в фазе FINAL
  - **Чтение реального веса через SimConnect**
  - **Ручной ввод веса и скорости в GUI**
- **FMS/GPS интеграция:**
  - Чтение маршрутных точек из плана полёта
  - Мониторинг прогресса по STAR
  - Отображение текущей и следующей точек
  - Расстояние и время до следующей точки
  - Поддержка схем захода из FMS
- **База данных аэропортов (NEW!):**
  - 8 крупных международных аэропортов
  - Готовые конфигурации ILS/VOR/NDB заходов
  - Поиск по ICAO, названию, городу
  - Легко расширяемая структура JSON
- **Графический диалог настройки (NEW!):**
  - Выбор из базы данных
  - Ручной ввод параметров
  - Предпросмотр информации о заходе
- Чтение навигационных данных (VOR/NDB)
- Чтение атмосферных данных (давление, температура, ветер)
- **Автоматическое выравнивание (Flare):**
  - Плавное увеличение тангажа (2.5° → 6.0°)
  - Снижение вертикальной скорости (600 → 100 fpm)
  - PD-контроллер для точного управления
  - Коррекция на ветер
  - Интеграция с vJoy для максимальной точности
- **Стабилизированный заход (Stabilized Approach):**
  - Автоматическая проверка критериев стабилизации (ICAO/FAA)
  - Мониторинг скорости, вертикальной скорости, крена, отклонений
  - Непрерывный мониторинг после высоты стабилизации
  - Автоматический уход на второй круг при нарушениях
  - Настройка для IMC (1000ft) / VMC (500ft)
- **Виртуальный джойстик (vJoy):**
  - Прямое управление рулями (элероны, руль высоты, руль направления)
  - Точная коррекция крена и тангажа
  - Автоматический расчёт управляющих входов
  - Гибридный режим (SimConnect + vJoy)
- **Работа с дальномером (DME):**
  - Использование DME для точного определения расстояния
  - Контрольные точки DME с проверкой высоты
  - Профили снижения по DME
  - DME Arc заходы (полёт по дуге)
  - DME Hold (ожидание на заданной дистанции)
  - Проверка точности DME
- **Автоматическая коррекция ветра:**
  - Расчёт угла сноса (drift angle)
  - Расчёт угла упреждения (crab angle)
  - Коррекция курса для компенсации бокового ветра
  - Коррекция вертикальной скорости при встречном/попутном ветре
  - Рекомендации по крену для компенсации бокового ветра
- **Проверка длины ВПП:**
  - Расчёт требуемой посадочной дистанции
  - Учёт ветра и состояния полосы
  - Предупреждения о недостаточной длине
  - Расчёт запаса безопасности
- **Графический интерфейс (GUI):**
  - Реальное время телеметрии
  - Панели навигации, стабилизации, выравнивания
  - Логи в реальном времени
  - Удобное управление заходом
- Автоматический расчёт параметров захода
- Управление автопилотом
- Фазы захода: Initial → Intermediate → Final → Landing

## Требования

- Microsoft Flight Simulator (2020 или новее)
- Python 3.7+
- SimConnect библиотека

## Установка

```bash
pip install SimConnect
```

### Опционально: vJoy для прямого управления

Для более точного управления установите vJoy:

1. Скачайте vJoy: https://sourceforge.net/projects/vjoystick/
2. Установите драйвер vJoy
3. Установите Python библиотеку:
```bash
pip install pyvjoy
```

Подробная инструкция: [docs/vjoy_setup.md](docs/vjoy_setup.md)

## Использование

### Графический интерфейс (рекомендуется)

1. Запустите Microsoft Flight Simulator
2. Загрузите полёт (самолёт должен быть в воздухе)
3. Запустите GUI:

```bash
cd C:/BAT/msfs_autoland
python gui.py
```

4. Нажмите "Connect" для подключения к MSFS
5. Нажмите "Start Approach" для настройки захода
6. В диалоге выберите:
   - **Из базы данных**: аэропорт, ВПП, тип захода (ILS/VOR/NDB)
   - **Ручной ввод**: все параметры вручную
7. Нажмите OK для начала захода
8. Следите за телеметрией и логами
9. При необходимости используйте "GO AROUND"

### Консольный режим

1. Запустите Microsoft Flight Simulator
2. Загрузите полёт (самолёт должен быть в воздухе)
3. Запустите программу:

```bash
cd C:/BAT/msfs_autoland
python main.py
```

4. Настройте параметры захода в коде (main.py, функция main())
5. Нажмите Enter для начала захода

## Конфигурация захода

### Из базы данных (GUI)

Просто выберите аэропорт из списка:
- **UUEE** - Шереметьево (Москва)
- **KJFK** - JFK (Нью-Йорк)
- **EGLL** - Heathrow (Лондон)
- **EGLC** - London City (крутая глиссада 5.5°!)
- **LFPG** - Charles de Gaulle (Париж)
- **EDDF** - Frankfurt (Франкфурт)
- **RJTT** - Haneda (Токио)
- **KLAX** - LAX (Лос-Анджелес)

### Ручной ввод (GUI или код)

Пример настройки ILS захода:

```python
from modules.airports_database import AirportsDatabase

db = AirportsDatabase()
config = db.get_approach_config('UUEE', '07C', 'ILS')
system.configure_approach(config)
```

Пример настройки VOR захода вручную:

```python
station = NavStation(
    name="Moscow VOR",
    frequency=11030000,  # 110.30 MHz в Hz
    latitude=55.5,
    longitude=37.5,
    type='VOR'
)

config = ApproachConfig(
    station=station,
    final_approach_course=270,  # курс посадки в градусах
    glideslope_angle=3.0,       # угол глиссады
    decision_height=200,         # высота принятия решения (футы)
    approach_speed=120,          # скорость захода (узлы)
    runway_elevation=500,        # превышение ВПП (футы)
    runway_length=8000,          # длина ВПП (футы)
    runway_width=150,            # ширина ВПП (футы)
    runway_threshold_lat=55.48,  # широта порога ВПП
    runway_threshold_lon=37.52   # долгота порога ВПП
)
```

## Фазы захода

1. **INITIAL** - Перехват курса посадки
2. **INTERMEDIATE** - Следование по курсу, снижение
3. **FINAL** - Точное следование по глиссаде
4. **LANDING** - Выравнивание и посадка

## Безопасность

⚠️ **ВАЖНО:**
- Система экспериментальная, используйте на свой риск
- Всегда следите за полётом и будьте готовы взять управление
- Рекомендуется тестировать на малых самолётах
- Проверяйте параметры захода перед использованием

## Логирование

Все действия записываются в `logs/autoland.log`

## Известные ограничения

- Требует ручной настройки параметров захода (или выбор из базы данных)
- Базовая логика управления
- База данных содержит ограниченное количество аэропортов (легко расширяется)
