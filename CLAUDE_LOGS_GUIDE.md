# Руководство: Совместная работа Claude с логами

## Обзор

Система позволяет Claude читать и анализировать логи работы программы, а все исправления вносятся совместно с пользователем.

## Как это работает

### 1. Программа создаёт логи

При работе программы автоматически создаются структурированные логи в папке `logs/`:
- `session_YYYYMMDD_HHMMSS.jsonl` - все события в JSON формате
- `session_YYYYMMDD_HHMMSS.log` - текстовые логи
- `errors_YYYYMMDD_HHMMSS.jsonl` - только ошибки

### 2. Claude читает и анализирует логи

**Команда:**
```bash
python claude_read_logs.py
```

**Что показывает:**
- 📊 Сводка (количество логов, ошибок, предупреждений)
- 🔍 Детали каждой ошибки:
  - Тип ошибки (KeyError, AttributeError, и т.д.)
  - Сообщение об ошибке
  - Количество повторений
  - Затронутые модули (например, `modules.telemetry`)
  - Затронутые функции (например, `get_data`)
  - Время первого и последнего появления
- ⚡ Проблемы производительности
- 💡 Рекомендации по исправлению

### 3. Совместное исправление

После анализа Claude:
1. Показывает найденные проблемы
2. Читает файлы с ошибками (используя Read tool)
3. Предлагает конкретные исправления
4. Применяет изменения с твоим подтверждением (используя Edit tool)

## Рабочий процесс

### Шаг 1: Запуск и тестирование программы

```bash
python gui.py
# Тестирование в MSFS
# Логи создаются автоматически
```

**Ты запускаешь:**

```bash
python claude_read_logs.py
```

### Шаг 4: Совместное исправление

**Диалог:**

```
Пользователь: "Клауд, исправь ошибку KeyError в telemetry.py"

Claude:
1. Читает файл: Read("modules/telemetry.py")
2. Находит проблему на строке 145: value = data['altitude']
3. Предлагает: "Нашёл проблему в telemetry.py:145. 
   Заменю data['altitude'] на data.get('altitude', 0) для безопасного доступа?"

Пользователь: "Да, исправь"

Claude:
4. Применяет: Edit(file_path="modules/telemetry.py", 
                   old_string="value = data['altitude']",
                   new_string="value = data.get('altitude', 0)")
5. Сообщает: "✅ Исправлено! Теперь код безопасно обрабатывает отсутствие ключа."
```

## Примеры использования

### Пример 1: Анализ после тестирования

```bash
# Ты
python gui.py
# ... тестирование в MSFS ...
# Закрытие программы

# Ты
python claude_read_logs.py

# Или просишь Claude
"Клауд, посмотри что в логах"

# Claude анализирует и показывает результаты
```

### Пример 2: Исправление конкретной ошибки

```
Ты: "Клауд, исправь ошибку в navigation.py"

Claude:
- Читает лог-анализ
- Находит: AttributeError в navigation.py, функция calculate_course
- Читает файл navigation.py
- Находит: result = obj.heading (без проверки на None)
- Предлагает: "Добавить проверку: if obj is not None: result = obj.heading"
- Ждёт подтверждения
- Применяет исправление
```

### Пример 3: Проверка производительности

```
Ты: "Клауд, есть ли проблемы с производительностью?"

Claude:
- Запускает: python claude_read_logs.py
- Показывает: "Обнаружена медленная операция telemetry_read: 125ms среднее"
- Читает код
- Предлагает оптимизацию
```

## Команды

### Для пользователя:

```bash
# Анализ последней сессии
python claude_read_logs.py

# Анализ конкретной сессии
python claude_read_logs.py --session 20260417_001530

# Полный анализ с генерацией отчёта
python analyze_logs.py --latest
```

### Для Claude:

Claude использует стандартные инструменты:
- **Bash** - запуск `python claude_read_logs.py`
- **Read** - чтение файлов с ошибками
- **Edit** - применение исправлений
- **Grep** - поиск проблемных мест в коде

## Структура папок

```
msfs_autoland/
├── logs/                          # Логи работы программы
│   ├── session_*.jsonl           # JSON логи
│   ├── session_*.log             # Текстовые логи
│   ├── errors_*.jsonl            # Только ошибки
│   └── analysis_*.md             # Отчёты анализа
├── modules/
│   ├── structured_logger.py      # Система логирования
│   └── log_analyzer.py           # Анализатор логов
├── claude_read_logs.py           # Чтение логов для Claude
└── analyze_logs.py               # Консольный анализ
```

## Преимущества

1. **Claude видит реальные ошибки** - не нужно гадать
2. **Конкретная информация** - модуль, функция, строка
3. **Совместная работа** - ты контролируешь все изменения
4. **Безопасность** - никаких автоматических изменений
5. **Прозрачность** - все действия Claude видны

## Типичный сценарий

1. **Тестирование:**
   ```bash
   python gui.py
   # Тестирование, возникают ошибки
   ```

2. **Анализ:**
   ```
   Ты: "Клауд, проанализируй логи"
   Claude: *запускает claude_read_logs.py*
   Claude: "Нашёл 3 ошибки: KeyError в telemetry.py, AttributeError в navigation.py..."
   ```

3. **Исправление:**
   ```
   Ты: "Исправь KeyError"
   Claude: *читает файл, находит проблему*
   Claude: "Нашёл на строке 145: data['altitude']. Заменить на data.get('altitude', 0)?"
   Ты: "Да"
   Claude: *применяет Edit*
   Claude: "✅ Исправлено!"
   ```

4. **Проверка:**
   ```bash
   python gui.py
   # Повторное тестирование
   python claude_read_logs.py
   # Проверка что ошибка исчезла
   ```

## Важно

- Все изменения вносятся только с твоим подтверждением
- Claude показывает что именно будет изменено
- Ты можешь отклонить любое предложение
- Логи помогают Claude понять реальные проблемы, а не гадать

3. Обсуди найденные проблемы
