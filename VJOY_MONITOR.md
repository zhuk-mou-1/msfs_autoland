# vJoy Monitor - Система мониторинга виртуального джойстика

**Дата:** 2026-04-17 15:15 (UTC+3)  
**Статус:** ✅ Реализовано

## Обзор

vJoy Monitor - система мониторинга виртуального джойстика в реальном времени. Отображает текущие значения всех осей, счётчики команд и предоставляет инструменты для тестирования.

## Возможности

### 1. Мониторинг в реальном времени
- **Текущие значения осей:** Aileron, Elevator, Rudder, Throttle
- **Визуальные индикаторы:** Цветные прогресс-бары для каждой оси
- **Счётчики команд:** Количество команд для каждой оси
- **Статус подключения:** Подключён/Отключён с цветовой индикацией

### 2. Визуализация
- **Прогресс-бары:** Зелёный (положительные значения), красный (отрицательные)
- **Центральная линия:** Показывает нулевую позицию
- **Синий индикатор:** Текущая позиция оси
- **Текстовые значения:** Точные числа от -1.00 до +1.00

### 3. Инструменты тестирования
- **Center All Axes:** Центрирование всех осей
- **Test Aileron:** Тест элеронов (влево → центр → вправо → центр)
- **Test Elevator:** Тест руля высоты (вниз → центр → вверх → центр)
- **Test Rudder:** Тест руля направления (влево → центр → вправо → центр)

```python
def get_status(self) -> dict:
    """Получить текущий статус vJoy для мониторинга"""
    return {
        'enabled': self.enabled,
        'device_id': self.device_id,
        'current_values': self.current_values.copy(),
        'command_count': self.command_count.copy(),
        'total_commands': sum(self.command_count.values())
    }
```

```python
def update_vjoy_monitor_panel(self):
    """Обновление панели мониторинга vJoy"""
    vjoy_status = self.system.virtual_joystick.get_status()
    
    # Обновление статуса
    if vjoy_status['enabled']:
        self.vjoy_status_var.set("Connected")
        self.vjoy_status_label.config(foreground='green')
    
    # Обновление значений осей
    for axis_name, value in vjoy_status['current_values'].items():
        self.vjoy_axis_vars[axis_name].set(f"{value:+.2f}")
        self._draw_axis_bar(canvas, value)
```

**_draw_axis_bar():**

```python
def _draw_axis_bar(self, canvas, value):
    """Рисование визуального индикатора оси"""
    # Центральная линия (серая)
    canvas.create_line(center_x, 0, center_x, height, fill='gray', width=2)
    
    # Бар (зелёный/красный)
    if value >= 0:
        color = 'green'  # Вправо/вверх
    else:
        color = 'red'    # Влево/вниз
    
    # Индикатор позиции (синий)
    canvas.create_line(indicator_x, 0, indicator_x, height, fill='blue', width=3)
```

### Интерпретация значений

**Aileron (Roll):**
- `-1.0` = Полный крен влево
- `0.0` = Нейтральное положение
- `+1.0` = Полный крен вправо

**Elevator (Pitch):**
- `-1.0` = Полный тангаж вниз
- `0.0` = Нейтральное положение
- `+1.0` = Полный тангаж вверх

**Rudder (Yaw):**
- `-1.0` = Полное рысканье влево
- `0.0` = Нейтральное положение
- `+1.0` = Полное рысканье вправо

**Throttle:**
- `-1.0` = Реверс (если поддерживается)
- `0.0` = Холостой ход
- `+1.0` = Полный газ

### Цветовая индикация

**Статус:**
- 🟢 Зелёный = Connected (vJoy работает)
- 🔴 Красный = Disconnected (vJoy не подключён)

**Прогресс-бары:**
- 🟢 Зелёный = Положительные значения (вправо/вверх)
- 🔴 Красный = Отрицательные значения (влево/вниз)
- 🔵 Синяя линия = Текущая позиция
- ⚫ Серая линия = Центр (0.0)

### Тестирование осей

**Center All Axes:**
- Устанавливает все оси в 0.0
- Используйте перед началом полёта
- Используйте после тестов

**Test Aileron:**
- Последовательность: -0.5 → 0.0 → +0.5 → 0.0
- Длительность: 2 секунды
- Проверяет работу элеронов

**Test Elevator:**
- Последовательность: -0.5 → 0.0 → +0.5 → 0.0
- Длительность: 2 секунды
- Проверяет работу руля высоты

**Test Rudder:**
- Последовательность: -0.5 → 0.0 → +0.5 → 0.0
- Длительность: 2 секунды
- Проверяет работу руля направления

## Диагностика проблем

### vJoy не подключается

**Проблема:** Status = Disconnected (красный)

**Решение:**
1. Проверьте установку vJoy Driver
2. Откройте vJoy Configure
3. Убедитесь что Device 1 включён (Enabled)
4. Проверьте что оси X, Y, Z настроены
5. Перезапустите AutoLand

### Оси не двигаются

**Проблема:** Значения остаются 0.00, счётчики не растут

**Решение:**
1. Проверьте что AutoLand подключён к MSFS
2. Проверьте что vJoy enabled в логах
3. Используйте кнопки Test для проверки
4. Проверьте MSFS Controls → vJoy Device

### Оси "прыгают"

**Проблема:** Значения меняются хаотично

**Решение:**
1. Проверьте что физический джойстик не конфликтует
2. Отключите другие программы управления (FSUIPC, SPAD.neXt)
3. Используйте Center All Axes
4. Проверьте настройки чувствительности в MSFS

### Счётчики не обновляются

**Проблема:** Total Commands = 0

**Решение:**
1. Убедитесь что AutoLand активно управляет самолётом
2. Начните заход на посадку (Start Approach)
3. Проверьте что use_vjoy = True в логах
4. Проверьте что vJoy не отключён в настройках

## Интеграция с другими системами

### Connection Monitor

vJoy Monitor работает независимо от Connection Monitor:
- Connection Monitor отслеживает SimConnect/WASM/L:Vars
- vJoy Monitor отслеживает виртуальный джойстик
- Оба могут работать одновременно

### Autothrottle

vJoy может управлять тягой через ось Throttle:
- Если `vjoy_throttle` включён
- Throttle ось будет показывать текущую тягу
- Счётчик throttle будет расти

### Flare Controller

Во время выравнивания (flare):
- Elevator будет активно меняться
- Значения будут в диапазоне -0.3 до +0.3
- Счётчик elevator будет быстро расти

## Производительность

### Накладные расходы

- **Обновление GUI:** Каждые 500мс
- **Сохранение значений:** ~0.01мс на команду
- **Рисование баров:** ~1-2мс на ось
- **Память:** ~1KB для хранения значений

### Оптимизации

1. **Ленивое рисование:** Бары перерисовываются только при изменении
2. **Кэширование Canvas:** Размеры вычисляются один раз
3. **Копирование словарей:** `copy()` для безопасности потоков

## Известные ограничения

1. **Требует vJoy Driver**
   - Должен быть установлен и настроен
   - Device 1 должен быть включён

2. **Только Device 1**
   - Мониторинг только первого устройства
   - Для других устройств нужна модификация

3. **Нет истории**
   - Показывает только текущие значения
   - Нет графиков изменения во времени

4. **Нет записи**
   - Нельзя сохранить сессию
   - Нельзя воспроизвести команды
